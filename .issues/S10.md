## What to build

C 端用户查看和管理自己的订单。

**小程序端**：
- 订单列表页：Tab 切换「待使用」「已核销」「已取消」
- 每条订单展示：场馆名称、场地号、日期、时段、价格、状态标签
- 订单详情：完整信息 + 状态时间线（预定时间 → 核销时间 或 取消时间）
- 取消功能：待使用状态的订单可取消（在免费取消时限内），取消后时段释放

**云函数 `cancelOrder`**：
- 校验订单属于当前用户、状态为「待使用」
- 校验是否在免费取消时限内（由场馆配置）
- 事务：订单状态改为「已取消」+ 对应 time_slot 改为「可预定」

## Acceptance criteria

- [ ] 订单列表按状态正确分类展示
- [ ] 订单详情展示完整信息和时间线
- [ ] 待使用订单可在取消时限内取消
- [ ] 取消后时段立即变为可预定
- [ ] 超过取消时限后不可取消（或按场馆规则处理）
